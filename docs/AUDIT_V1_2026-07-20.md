## Three bugs fixed in this audit

### Bug 1 — dead expression in lineup builder
- **File:** `ud_edge/matcher.py:510`
- **Symptom:** `n_entries * n_legs` evaluated, value discarded; next line uses `len(ranked) // n_legs`.
- **Reproduction:** `read_file matcher.py:510` shows the stray statement.
- **Fix:** removed the line.
- **Risk:** zero — expression had no side effects, no tests asserted on it.

### Bug 2 — broken P1 #8 test suite silently dropped by pytest
- **File:** `tests/test_audit_p1_propline_cache_fallback.py:25`
- **Symptom:** `from tests.test_propline import SAMPLE_EVENT` raised `ModuleNotFoundError: No module named 'tests.test_propline'`. Pytest reported `ERROR collecting` (not FAIL), so the entire 8-function / 190-line P1 #8 regression suite was **silently skipped** in CI. `ruff check` separately flagged `parse_event_odds` as imported but unused.
- **Fix:** added empty `tests/__init__.py` so `tests` becomes a package; removed the unused import.
- **Reproduction (before):** `pytest --collect-only tests/test_audit_p1_propline_cache_fallback.py` → `ERROR collecting tests/...` (1 error).
- **Reproduction (after):** `pytest -q tests/test_audit_p1_propline_cache_fallback.py` → 6 passed.

### Bug 3 — stale test fixture from Wave 2B
- **File:** `tests/test_audit_p1_line_tolerance_plumbing.py:84`
- **Symptom:** fixture used `scheduled_at="2026-07-20T20:00:00+00:00"`. Wave 2B added `reject_started=True` to `rank_legs` (P1 #5); the date is now in the past, so the leg is filtered before fuzzy matching runs. Test failed with `[matcher] skipped 1 started → 0 +EV legs kept`.
- **Fix:** replaced the hardcoded date with `(datetime.now(timezone.utc) + timedelta(days=2)).isoformat()` so the test is robust against clock drift while still exercising the fuzzy-match path.
- **Reproduction (after):** 1 was failing, now 1 passes; full file = 7 passed.

## Known limitations (will not fix without user direction)

- **No external live data.** PrizePicks/Sleeper public endpoints return 403; Pinnacle/DK/FD props require paid `OddsAPI` / PropLine / `SportsGameOdDS` keys. The bot runs from disk cache + ESPN injury feed (no auth) + manual CSV (`data/sharp_lines.csv`) when keys are absent.
- **No settlement pipeline.** Settle count = 0/50. `safety_gate.is_research_mode()` stays True. All recommendation labels render as `🟡 RESEARCH ESTIMATE`.
- **No cron.** Per Wave 0 contract, no scheduled job. Operator runs `python -m ud_edge --once` or `--poller` manually.
